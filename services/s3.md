# S3 (Simple Storage Service)

Object storage. Think of it as an infinite-capacity hard drive on AWS — files (called **objects**) live inside named containers (called **buckets**).

**What it's good for:**
- Static website hosting (HTML/CSS/JS) → served via CloudFront
- Backups, logs, archived data
- Lambda code zips, container images (via ECR which uses S3 underneath)
- CDK deploy artifacts (the `cdk-hnb659fds-assets-...` bucket you saw after `cdk bootstrap`)

**What it's NOT good for:**
- Databases (use DynamoDB or RDS)
- Tiny key-value lookups in hot paths (S3 latency ~50-200ms; DynamoDB ~5-10ms)
- Filesystem semantics like rename/append — every "edit" is a full re-upload

---

## Concepts (quick)

| Term | Meaning |
|---|---|
| **Bucket** | Top-level container. Lives in one AWS Region. Name is globally unique. |
| **Object** | A single file inside a bucket. Up to 5 TB each. |
| **Key** | The "path" of an object inside the bucket (e.g. `images/logo.png`). S3 is flat — folders are just key prefixes. |
| **Prefix** | A logical "folder" (any string ending in `/`). |
| **Region** | AWS data center geography. Affects latency + pricing. Use `us-east-1` unless you have a reason. |
| **OAC** | Origin Access Control. Lets CloudFront read a private bucket without exposing it to the public internet. |

## Creating a bucket — Console

1. Top search bar → **S3** → **Create bucket**
2. **Bucket namespace:** **Global namespace** (compatible with all tutorials/tools; Account Regional is newer but introduces friction)
3. **Bucket name:** `<project>-<purpose>-<account-id>` — e.g. `order-d20-frontend-277207922271`
   - **Globally unique** across all AWS accounts on Earth
   - Lowercase, no underscores, 3-63 chars
   - Account-ID suffix is a convention for guaranteeing uniqueness
4. **Region:** **US East (N. Virginia) us-east-1**
5. **Object Ownership:** **ACLs disabled (recommended)** — leave default
6. **Block Public Access:** **✅ Block ALL public access** (all 4 checkboxes on)
   - Old tutorials say to make buckets public for static hosting — **don't.** Use CloudFront + OAC.
7. **Bucket Versioning:** **Disable** unless you need recoverable deletes for user data
8. **Tags:** add `Project=<your-project>` (lets you split billing by project later)
9. **Default encryption:** **SSE-S3** (free, default)
10. **Create bucket**

## Creating a bucket — CLI

```bash
aws s3 mb s3://<bucket-name> --region us-east-1
```

That's it. To match the console defaults (block public access, encryption), do:

```bash
aws s3api create-bucket --bucket <bucket-name> --region us-east-1

# Block all public access
aws s3api put-public-access-block --bucket <bucket-name> \
  --public-access-block-configuration \
  "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```

> **Note:** For regions other than `us-east-1`, `create-bucket` requires `--create-bucket-configuration LocationConstraint=<region>`.

## Uploading files

### Console (drag and drop) — good for one-off uploads

1. Open the bucket → **Upload** button
2. Drag files/folders into the drop zone OR click **Add files / Add folder**
3. Leave **Properties** and **Permissions** at defaults (object inherits bucket settings)
4. **Upload**

**Limits:**
- Up to ~160 GB per upload session via browser
- Browser is single-threaded — slow for hundreds of small files
- Re-uploading the same file overwrites it (no diff)

### CLI sync — what you'll actually use

```bash
aws s3 sync <local-dir>/ s3://<bucket-name>/ --delete
```

**Flags:**
- `--delete` — remove files in S3 that no longer exist locally (clean mirror)
- `--exclude "*.DS_Store" --exclude "node_modules/*"` — skip junk
- `--dryrun` — preview without uploading (always use before `--delete`)
- `--acl public-read` — make objects publicly readable (only if bucket is public; usually don't)

**Common use:**
```bash
# Deploy frontend
aws s3 sync ui/static/ s3://order-d20-frontend-277207922271/ --delete

# Dry run first
aws s3 sync ui/static/ s3://order-d20-frontend-277207922271/ --delete --dryrun
```

### CLI single-file commands

```bash
# Upload one file
aws s3 cp localfile.html s3://my-bucket/path/in/bucket/file.html

# Download one file
aws s3 cp s3://my-bucket/path/file.html ./localfile.html

# Move (cp + delete original)
aws s3 mv localfile.html s3://my-bucket/path/file.html
```

## Viewing / listing files

### Console
Click into the bucket → file tree view, with size, last modified, storage class.

### CLI
```bash
# List bucket root
aws s3 ls s3://my-bucket/

# List with sizes (human-readable) + recursive
aws s3 ls s3://my-bucket/ --recursive --human-readable --summarize

# List all buckets in account
aws s3 ls
```

`PRE css/` in `aws s3 ls` output = "prefix" = folder. Confusing the first time you see it.

## Deleting

### Console
Select object(s) → **Delete** button → type "permanently delete" → confirm.

### CLI
```bash
# Delete one object
aws s3 rm s3://my-bucket/file.html

# Delete a whole folder
aws s3 rm s3://my-bucket/folder/ --recursive

# Empty entire bucket (CANNOT be undone)
aws s3 rm s3://my-bucket/ --recursive

# Delete the bucket itself (must be empty first)
aws s3 rb s3://my-bucket
# Or force-empty + delete:
aws s3 rb s3://my-bucket --force
```

> **`--force` deletes everything in the bucket and the bucket itself, with no confirmation prompt. Triple-check the bucket name.**

## Security: the 3 ways to expose S3 (in order of safety)

| Pattern | Use case | Risk |
|---|---|---|
| **Private bucket + CloudFront OAC** | Web frontend hosting (this project) | Lowest. CloudFront only entity that can read. |
| **Private bucket + presigned URLs** | Time-limited user file downloads | Low. URLs expire. |
| **Public bucket policy** | Almost nothing modern | High. Easy to leak. The default AWS data breach. |

**Always use Block Public Access ON unless you have a specific, audited reason not to.**

### Bucket policy basics

A bucket policy is a JSON document attached to the bucket that says who can do what. CloudFront OAC generates one for you (allows `s3:GetObject` only when the request comes from your specific CloudFront distribution).

To view/edit:
- **Console:** Bucket → **Permissions** tab → **Bucket policy** → Edit
- **CLI:**
  ```bash
  aws s3api get-bucket-policy --bucket <bucket-name>
  aws s3api put-bucket-policy --bucket <bucket-name> --policy file://policy.json
  ```

## Common operations cheat sheet

```bash
# Create bucket
aws s3 mb s3://my-bucket --region us-east-1

# Upload directory (sync, with delete)
aws s3 sync ./dist/ s3://my-bucket/ --delete

# Download whole bucket
aws s3 sync s3://my-bucket/ ./local-copy/

# List all buckets
aws s3 ls

# List bucket contents recursively
aws s3 ls s3://my-bucket/ --recursive --human-readable --summarize

# Check bucket size
aws s3 ls s3://my-bucket/ --recursive --summarize | tail -2

# Empty + delete bucket
aws s3 rb s3://my-bucket --force

# Get bucket region
aws s3api get-bucket-location --bucket my-bucket

# Get bucket policy
aws s3api get-bucket-policy --bucket my-bucket --output text | jq .
```

## Pricing (free tier + beyond)

**Free tier (12 months for new accounts):**
- 5 GB storage
- 20,000 GET requests/month
- 2,000 PUT/COPY/POST requests/month
- 100 GB egress/month (shared with all AWS services)

**After free tier (us-east-1, S3 Standard):**
- **Storage:** ~$0.023/GB/month
- **PUT/POST/COPY:** ~$0.005 per 1,000 requests
- **GET:** ~$0.0004 per 1,000 requests
- **Egress to internet:** ~$0.09/GB (this is the expensive one — CloudFront is much cheaper)
- **Egress to CloudFront:** Free ← why we always front S3 with CloudFront

**For Order D20 at 10 users:** essentially $0/month. Static site is a few MB; ~thousands of requests/month.

**Watch out:**
- Lifecycle policies (move old logs to Glacier) become important at TB scale
- Cross-region replication is **expensive** (egress charges apply)
- Versioning doubles storage cost if every file is modified frequently

## Common gotchas

**"Access Denied" when fetching from CloudFront**
Bucket policy not yet updated with OAC permissions. CloudFront shows the policy to paste — apply it under Bucket → Permissions → Bucket Policy.

**Bucket name "already exists"**
Names are global. Add your account ID as a suffix, or pick something more specific.

**Can't delete bucket: "BucketNotEmpty"**
Use `aws s3 rb s3://bucket --force` or empty it first in the console.

**`PRE foldername/` in CLI output**
That's S3's way of saying "folder/prefix." There are no real folders in S3 — just keys with `/` in the name.

**Re-upload of same file with `cp` doesn't update timestamp / browser cache**
Browsers cache static files. After uploading new content, you need to **invalidate CloudFront** (Distribution → Invalidations → Create → `/*`) or version the filenames.

**`aws s3 sync` deletes files from S3 I didn't expect**
`--delete` removes anything in S3 not present locally. Run `--dryrun` first. Especially burns you the first time you sync from the wrong source folder.

**Bucket region locked**
You can't change a bucket's region after creation. To "move" a bucket, you copy contents to a new bucket in the target region, then delete the old one.

**Block Public Access overrides everything**
Even a bucket policy that grants public access does nothing if Block Public Access is on. This is the safest default; turn it off only when you've thought about it.

## When to reach for S3 vs. alternatives

| Need | Use |
|---|---|
| Static web frontend | **S3 + CloudFront** |
| User uploads / large files | **S3** (with presigned URLs for direct browser upload) |
| Application data (records, JSON) | **DynamoDB** (NoSQL) or **RDS/Aurora** (SQL) |
| Caching session data, hot lookups | **ElastiCache** or **DynamoDB** |
| Shared filesystem across EC2/Lambda | **EFS** (not S3 — S3 isn't a filesystem) |
| Container images | **ECR** |

S3 is the default answer for "where do I put files," but resist using it as a database — every read is a full GET, latency is high vs. DDB.
